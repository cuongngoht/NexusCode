# Nexus OOP/OOD Policy

Review for:
- encapsulation
- abstraction
- cohesion
- coupling
- SOLID
- dependency inversion
- interface segregation
- replaceability
- testability
- class responsibility
- method size
- constructor dependency count
- public API stability

Avoid:
- God Object
- Large Class
- Long Method
- Feature Envy
- Shotgun Surgery
- Data Clump
- Primitive Obsession
- Switch/if mode explosion
- Interface with no boundary value
- Inheritance when composition is simpler
