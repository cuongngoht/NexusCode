# Write Tests Skill

Use this workflow when adding or improving tests.

Focus on:
- Unit tests
- Integration tests
- Edge cases
- Regression coverage
- Mocking external dependencies
- Verifying user-visible behavior

Rules:
- Prefer tests close to the changed code.
- Cover happy path, failure path, and edge cases.
- Do not over-mock core logic.
- Name tests clearly.

Output:
- Test strategy
- Test files to add or update
- Test cases
- Mocking plan
- Commands to run
