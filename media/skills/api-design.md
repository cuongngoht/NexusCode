# API Design Skill

Use this workflow when designing internal or external APIs.

Focus on:
- Clear interfaces
- Stable types
- Minimal surface area
- Backward compatibility
- Error handling
- Naming consistency

Rules:
- Prefer explicit types.
- Keep APIs small.
- Separate transport shape from domain model when useful.
- Include examples.

Output:
- Proposed API
- Types/interfaces
- Data flow
- Error cases
- Migration notes
