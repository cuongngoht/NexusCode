# Security Review Skill

Use this workflow when checking for security risks.

Focus on:
- Path traversal
- Command injection
- Secret leakage
- Unsafe file access
- Trust boundaries
- Dangerous defaults
- User-controlled input

Rules:
- Report only grounded risks.
- Explain why each issue matters.
- Suggest practical fixes.
- Avoid vague security warnings.

Output:
- Security summary
- Issues found
- Severity
- Suggested fix
- Tests to add
