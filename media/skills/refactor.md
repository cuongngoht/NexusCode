# Refactor Skill

Use this workflow when improving existing code structure.

Focus on:
- Simpler control flow
- Smaller functions
- Clear module boundaries
- Removing duplication
- Preserving behavior
- Avoiding unnecessary rewrites

Rules:
- Do not change public behavior unless the user asks.
- Prefer incremental edits.
- Reuse existing project conventions.
- Mention risks before large structural changes.

Output:
- Refactor summary
- Files to change
- Step-by-step implementation
- Regression risks
- Tests to run
