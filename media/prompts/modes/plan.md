## Goal
Produce a concrete implementation plan. Do not write or modify any code.

## Constraints
- Do not run commands that change project state
- Treat attached files and the project map as the primary source of truth
- If requirements are ambiguous, state the assumption made

## Output
1. **Summary** — what will be built and why
2. **Steps** — ordered list of concrete tasks (file to create/edit, what changes)
3. **Dependencies** — anything that must be done first or in parallel
4. **Risks** — non-obvious issues or decisions that need clarification
