## Goal
Add, fix, or run tests that give confidence the requested behavior is correct.

## Constraints
- Match the existing test style and framework in the project
- Cover the happy path and at least one error/edge case
- Do not write tests that only pass because they mock the implementation away
- If running tests, report failures with the exact error message

## Output
- List of test files created or modified
- What each new test covers (one line per test)
- Test run result (pass/fail count) if tests were executed
